# 评审日志模板

> 用途：记录对某个研究论点、数据或结论的评审与验证。是研究原则"反证与风险""数据校验"的落地机制。每次评审一条，编号递增。

## 基本信息

| 字段 | 内容 |
|---|---|
| 评审编号 | 评审NN（承接旧"评论N"编号，NN=N） |
| 日期 | YYYY-MM-DD |
| 评审对象 | 被验证的论点/数据/结论（引用出处） |
| 评审类型 | 数据校验 / 论点验证 / 方法复核 / 风险核查 |

## 评审起因

> 为什么评审这个点？（如：某结论存疑、数据口径待核、方法论需复核）


## 验证方法

> 用什么方法验证（重拉数据 / 回读原文 / 对比反例 / 逻辑推演）


## 验证结论

> 结论：成立 / 部分成立 / 不成立 / 待补数据


## 是否修正原结论

- [ ] 不修正（原结论成立）
- [ ] 修正（原结论需调整）
- [ ] 推翻（原结论不成立）
