# 市场新趋势研究框架

> 研究对象不是“热点罗列”，而是海外手游市场中可被案例和数据验证、对公司研发与发行战略有参考价值的新变化。

## 1. 核心研究问题

### 一级问题

```text
海外手游市场正在出现哪些新的产品、玩法、发行、商业化或用户行为变化？
哪些变化已经具备商业验证？
哪些变化对公司未来立项、发行、买量和组织能力建设有实际价值？
```

### 具体问题

1. 哪些新玩法或融合方向出现多产品复现？
2. 哪些产品的增长不是偶发，而是代表结构性变化？
3. 哪些趋势适合公司研发进入，哪些只适合发行/买量借鉴？
4. 哪些趋势是短期素材噱头，哪些能支撑长线产品？
5. 哪些海外市场最先出现变化，美国、日本、韩国、欧洲还是新兴市场？
6. 新趋势对公司现有品类、在研项目和发行策略有什么影响？

## 2. 趋势扫描维度

### A. 产品与玩法趋势

| 方向 | 关注问题 | 例子/待补充 |
|---|---|---|
| 轻度玩法中度化 | 轻度核心 + 长线养成/赛季/装备是否成为常态 | 待补充 |
| 混合休闲新形态 | 超休闲衰退后，哪些混合玩法有收入能力 | 待补充 |
| 肉鸽机制扩散 | 肉鸽是否作为内容生成与买量表达工具扩散 | 见肉鸽研究框架 |
| 玩法广告化 | 素材先火，产品再承接的链路是否稳定 | 待补充 |
| 低成本内容消耗 | 通过随机、UGC、关卡变体降低内容成本 | 待补充 |
| 社交/多人轻竞技 | 派对、异步对抗、轻社交是否回升 | 待补充 |
| 高 ARPU 小众题材 | 小众但高付费题材是否值得关注 | 待补充 |
| Web/社区驱动 | Discord、Web、UGC 社区是否影响手游增长 | 待补充 |

### B. 发行与买量趋势

| 方向 | 关注问题 | 数据/证据 |
|---|---|---|
| 创意素材驱动 | 增长是否由新素材样式带动 | Creative Gallery、SOV、下载峰值 |
| 买量渠道变化 | 哪些网络/国家变化明显 | Ad Network Analysis |
| 软启动策略 | 产品是否更长时间灰度测试 | 版本记录、国家分布 |
| 国家迁移 | 某玩法是否先在新兴市场验证再进入 Tier 1 | 国家下载/收入拆分 |
| 发行商打法 | 哪些发行商频繁验证新品类 | publisher/app 数据 |

### C. 商业化趋势

| 方向 | 关注问题 | 证据 |
|---|---|---|
| 混合变现 | IAA + IAP 是否成为更多品类默认配置 | 收入结构、产品体验 |
| 订阅/通行证 | 中轻度产品是否提高长期付费 | IAP、版本活动 |
| 低价礼包/高频付费 | 付费设计是否更轻量化 | IAP、收入曲线 |
| 广告变现压力 | 广告驱动产品收入质量如何 | 下载/收入/RPD 对比 |

### D. 用户与内容趋势

| 方向 | 关注问题 | 证据 |
|---|---|---|
| 短周期内容消费 | 用户是否更偏好快速反馈与高频变化 | 留存、评论、版本节奏 |
| 构筑/选择偏好 | 用户是否偏好“选择—变强—重开”的体验 | 玩法拆解、评论 |
| 观看式玩法 | 素材可看性是否影响下载 | 素材、SOV、榜单变化 |
| 社区参与 | 用户是否参与二创、攻略、分享 | Reddit/Discord/YouTube/TikTok |

## 3. 趋势成立标准

一个市场新趋势要进入战略会讨论，至少满足以下条件：

### 必须满足

1. 有明确的趋势定义，不是模糊热词。
2. 至少有 2-3 个代表案例或一个强案例 + 多个跟随信号。
3. 有数据支撑：下载、收入、排名、活跃、买量或社区信号之一。
4. 能说明对公司的具体价值：立项、迭代、发行、买量、组织能力。

### 强建议满足

1. 有反例或风险分析。
2. 能区分短期买量现象和长期产品趋势。
3. 能判断适合的国家/用户/品类。
4. 能提出下一步验证动作。

## 4. 趋势分级

| 等级 | 定义 | 建议动作 |
|---|---|---|
| A | 有多产品复现 + 数据验证 + 公司可执行 | 进入专题研究/战略会重点页 |
| B | 有明确信号但验证不足 | 建立观察池，补数据和案例 |
| C | 目前只是热点或孤例 | 低频观察，不投入主要资源 |
| D | 与公司业务关联弱或证据不足 | 暂不跟进 |

## 5. Sensor Tower 数据使用方式

### 趋势验证常用问题

| 问题 | 数据能力 |
|---|---|
| 是否真的增长 | 下载/收入时间序列 |
| 是否只是买量冲榜 | 下载来源、广告 SOV、榜单峰值持续性 |
| 是否有收入质量 | 收入、RPD、国家收入结构 |
| 是否多市场成立 | 国家拆分 |
| 是否多产品复现 | 产品池批量查询 |
| 是否有活跃支撑 | active_users |
| 是否有素材扩散 | creatives、network_analysis |

### 常用 endpoint

| Endpoint | 用途 |
|---|---|
| `/v1/{os}/sales_report_estimates` | 下载/收入趋势 |
| `/v1/{os}/apps` | 产品元数据 |
| `/v1/{os}/usage/active_users` | 活跃用户 |
| `/v1/{os}/category/category_history` | 类别排名历史 |
| `/v1/{os}/ad_intel/creatives` | 广告素材 |
| `/v1/{os}/ad_intel/network_analysis` | 买量 SOV |
| `/v1/{os}/downloads_by_sources` | 下载来源 |
| `/v1/{os}/app_update/get_app_update_history` | 版本更新时间线 |

## 6. 研究输出结构

每个趋势专题建议输出：

```text
1. 一句话结论
2. 趋势定义
3. 代表案例
4. 数据证据
5. 成立逻辑
6. 反证与风险
7. 对公司的机会
8. 建议动作
9. 后续验证
```

## 7. 当前候选趋势池

> 本节趋势随研究进展回填。状态：待验证 / 已验证 / 已推翻 / 已排除 + 证据链接。回填见 `你的完整报告`。

| 趋势 | 状态 | 验证结论 | 证据 |
|---|---|---|---|
| 轻度玩法中度化 | 已验证（在研内） | 示例产品A/B 等验证，但与公司在研方向重叠，非新方向 | 周报信号聚类 |
| 肉鸽机制泛化 | 已验证（在研内） | 见赛道框架示例，属公司在研方向深化 | 你的肉鸽研究报告 |
| 素材先行/玩法广告化 | 已排除 | 是买量方法论非品类方向 | 你的完整报告 §2.5 |
| 混合休闲收入升级 | 已验证（B） | 与中度化同源，RPD/国家分布已验证 | 你的完整报告 §3 |
| 低成本内容生成机制 | 待验证 | 需随机/UGC/关卡变体对留存影响数据 | — |
| 社交/轻多人回潮 | 已验证（A） | 示例产品C $XX->YY M 复苏、产品D 翻倍（示例占位） | 你的完整报告 §4 |
| AI/UGC 手游机会 | 已推翻（C） | ST 无商业规模验证，暂不投入 | 你的完整报告 §6.3 |

## 8. 历史资料接入规则

后续用户提供历史研究资料时，先归档到：

```text
00_输入资料/历史研究/
```

再按以下步骤处理：

1. 提取已有结论
2. 判断是否仍有效
3. 映射到趋势扫描维度
4. 标记缺失证据
5. 用 Sensor Tower 或外部资料补证
6. 沉淀到 `你的研究目录/结论沉淀/`

## 9. 战略会判断口径

每个趋势最终要落到以下四类之一：

| 结论类型 | 含义 |
|---|---|
| 立即推进 | 已有足够证据，建议形成项目/专题动作 |
| 小规模验证 | 有价值但不确定，建议低成本测试 |
| 持续观察 | 信号存在但不足，建立监控池 |
| 暂不投入 | 证据弱、不可复制或与公司不匹配 |
