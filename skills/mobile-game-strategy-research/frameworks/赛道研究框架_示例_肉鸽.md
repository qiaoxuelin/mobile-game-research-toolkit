# 肉鸽赛道研究框架

> 研究对象不是单一产品，而是“肉鸽机制在海外手游中的商业化扩散与可执行机会”。本框架用于承接历史研究资料、Sensor Tower 数据、竞品拆解与战略结论。

## 1. 核心研究问题

### 一级问题

```text
肉鸽机制在海外手游市场中，哪些融合方向已经被商业验证？
哪些只是短期概念或买量噱头？
哪些方向适合我们公司研发、发行或现有项目迭代？
```

### 需要回答的具体问题

1. 肉鸽机制在哪些手游品类中扩散最明显？
2. 哪些产品已经证明有下载、收入或活跃表现？
3. 成功来自肉鸽机制本身，还是来自题材、美术、买量、IP、平台红利？
4. 肉鸽在中轻度手游中承担的是：新鲜感、长线内容、付费驱动，还是素材卖点？
5. 哪些融合方式研发成本可控、内容消耗可控、适合海外发行？
6. 对公司来说，应该关注立项机会、系统借鉴、买量素材，还是现有项目迭代？

## 2. 肉鸽方向分类体系

### A. 按玩法融合分类

| 方向 | 说明 | 代表案例 | 研究优先级 |
|---|---|---|---|
| 肉鸽 + 路径塔防 | 路线选择、关卡事件、塔防构筑，如 Raid Rush 类 | Raid Rush 等 | A |
| 肉鸽 + 幸存者-like | 局内随机技能、割草、生存波次 | 示例幸存者-like 品 等 | A |
| 肉鸽 + 休闲射击 | 轻操作射击 + 随机构筑 | 待补充 | A/B |
| 肉鸽 + 卡牌/构筑 | 牌组、遗物、路线、事件 | Slay the Spire-like 移动化 | B |
| 肉鸽 + RPG/养成 | 局内随机 + 局外角色/装备成长 | 待补充 | A/B |
| 肉鸽 + 放置/轻度养成 | 随机选择弱化，偏长线数值成长 | 待补充 | B |
| 肉鸽 + 益智/解谜 | 随机道具、关卡变体、构筑增强 | 待补充 | B/C |
| 肉鸽 + 策略/SLG 局部系统 | 事件、探索、战斗构筑作为模块 | 待补充 | C |

### B. 按肉鸽机制强度分类

| 类型 | 判断标准 | 研究含义 |
|---|---|---|
| 强肉鸽 | 路线、事件、构筑、随机性、失败重开都构成核心循环 | 适合研究核心玩法与长线内容结构 |
| 中肉鸽 | 局内随机技能/道具明显，但局外成长决定长期目标 | 更适合手游商业化，重点看留存与付费 |
| 弱肉鸽 | 只有随机奖励、随机关卡或素材包装 | 谨慎判断，可能只是噱头 |
| 肉鸽素材化 | 买量素材强调选择/随机/变强，但实际产品不一定是肉鸽 | 重点看素材转化，不直接代表玩法趋势 |

### C. 按商业化角色分类

| 肉鸽机制角色 | 说明 | 关注指标 |
|---|---|---|
| 核心玩法 | 肉鸽决定主要体验 | 下载、收入、留存迹象、版本迭代 |
| 长线内容生成器 | 用随机组合降低内容消耗压力 | 活跃、版本节奏、内容更新压力 |
| 付费驱动器 | 通过构筑、角色、装备、局外养成推动付费 | RPD、收入曲线、IAP 结构 |
| 买量素材卖点 | 随机选择、爽感成长适合短视频表达 | 创意数量、SOV、下载来源 |
| 中度化桥梁 | 把轻度玩法引向更长线系统 | 下载转收入、DAU/MAU、流失风险 |

## 3. 判断标准

### 趋势成立标准

一个肉鸽方向要被视为“趋势”，至少满足以下 3 条中的 2 条：

1. 多个产品复现，而不是单一爆款。
2. 至少在一个核心海外市场出现下载/收入验证。
3. 肉鸽机制与增长之间存在合理因果，而非纯 IP/买量/渠道驱动。
4. 后续产品或素材在模仿，说明行业在扩散。
5. 玩法结构具备可复制性，而不是高度依赖特定团队或 IP。

### 战略价值判断

| 维度 | 问题 |
|---|---|
| 市场空间 | 是否有足够下载/收入规模？是否只适合小众？ |
| 增长质量 | 下载、收入、活跃是否匹配？是否只是短期买量？ |
| 研发可行性 | 内容量、数值、关卡、构筑复杂度是否可控？ |
| 发行可行性 | 是否适合买量表达？目标国家是否清晰？ |
| 差异化 | 我们进入是否还能做出差异？ |
| 与现有项目关联 | 是否能服务公司现有或在研项目？ |
| 风险 | 同质化、长期内容压力、付费弱、玩法疲劳等 |

## 4. 证据要求

每个重要判断至少需要以下证据组合：

### 必需证据

- Sensor Tower 下载/收入数据
- 代表产品列表
- 产品机制拆解
- 国家/市场分布

### 强建议证据

- 类别排名变化
- 活跃用户或留存迹象
- 广告素材/SOV/下载来源
- 版本更新节奏
- 玩家评论或社区反馈
- 反例产品
