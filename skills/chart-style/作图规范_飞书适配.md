# 咨询风格图表作图规范

> 本工程所有面向战略会的图表（飞书文档 SVG 画板）一律按此规范执行。
> 参考自 `chart-style-extracted/`，结合飞书 whiteboard 约束精简为可执行清单。

---

## 一、核心原则（不可违背）

1. **洞察标题** — 左上角粗体，一句话陈述结论（如"X 驱动 73% 增长"）。**禁用**描述性标题（如"X 按 сегмент 分布"）。
2. **最小墨水** — 无背景填充、无 3D、无阴影、无渐变、无装饰边框。白底 `#FFFFFF`。
3. **仅横向网格** — `#D9D9D9` 0.5px，**无纵向网格、无纵向轴线**。
4. **直接标签** — 数据直接标在图上（线尾、柱顶），**避免图例**。
5. **Source 行** — 底部左侧 8pt `#808080`：`Source: <来源>; 分析`。
6. **字体** — `font-family: 'Arial', 'Helvetica Neue', sans-serif`。

---

## 二、配色（严格）

| 角色 | 名称 | Hex | 用途 |
|---|---|---|---|
| 主数据 | Primary Navy | `#002060` | 高亮线/主柱/标题 |
| 次要 | 深灰 | `#404040` | 次柱 |
| 第三 | 中灰 | `#808080` | 支撑线 |
| 弱化 | 浅灰 | `#BFBFBF` | 背景线/ghost |
| 网格 | Grid Grey | `#D9D9D9` | 仅网格 |
| 正文 | Body | `#595959` | 轴标签/注释 |
| 标题 | Headline | `#000000` | 洞察标题 |
| accent（每图最多1个） | 蓝 `#0070C0` / 橙 `#FF6600`（负）/ 绿 `#00B050`（正） | — | 突出单一洞察 |

**规则**：每图最多 1 个 accent；禁红绿同图；主数据填充 opacity ≥ 0.5。

---

## 三、线图规则

- **1 条高亮线**：Navy `#002060`，2.5px —— 对应洞察重点
- **2–4 条支撑线**：`#808080`，1.5px，opacity 0.6
- **其余线**：`#BFBFBF`，1px，opacity 0.4
- 点标记：同色实心圆 r=4
- 直接标在线尾（产品名 + 占比/终值），**不用图例**
- 多线区分困难时（如 10 条），可破例用全彩，但必须加洞察标题 + Source 行

---

## 四、布局（ViewBox 800×480 横版）

```
┌──────────────────────────────────────────┐  y=0
│ [洞察标题]                       [Tag]   │  y=20–55
│──────────────────────────────────────────│  y=58 divider
│                                          │
│            CHART AREA                    │  y=70 to y=430
│                                          │
│──────────────────────────────────────────│  y=435
│ Source: ...                              │  y=465
└──────────────────────────────────────────┘  y=480
```

- 左边距 60，右 40，上 70，下 50
- 图区：x=60–760，y=70–430
- 标题：x=60 y=35，15px bold `#000000`，左对齐，最大宽 600（超则 y=50 折行）
- divider：`<line x1="60" y1="58" x2="740" y2="58" stroke="#D9D9D9" stroke-width="0.8"/>`
- Source：x=60 y=465，8px `#808080`
- 轴标签 10px `#595959`；数据标签 10px

---

## 五、飞书 whiteboard SVG 约束

- 插入格式：`<fragment><whiteboard type="svg">完整自包含 SVG</whiteboard></fragment>`
- SVG 必须含 `xmlns="http://www.w3.org/2000/svg"` 和 `viewBox`
- **不引用**外部图片/脚本/远程资源；文字用 `<text>`（非 `<path>`）
- svg-parser 识别 `text/rect/line/path/circle/polyline` 转可编辑节点；`filter`（drop-shadow 等）降级为内嵌图片（渲染 OK，不可编辑）—— 慎用
- 插入用 `lark-cli docs +update --command block_insert_after --content @文件`（@file 避免引号问题）
- 插入后 fetch 显示 `<whiteboard token="...">` 是**正常**的（飞书把 inline svg 转画板资源）
- **替换图表**：删除旧 wb + 重新 block_insert_after；**不要用 block_replace**（对 inline svg 不稳定，会变空 token）
